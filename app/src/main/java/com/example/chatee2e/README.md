
## Технічні компоненти рівня застосунку

### MainActivity.kt
Єдина точка входу в інтерфейс (Single Activity Architecture). Налаштовує `setContent`, ініціалізує навігаційний контролер та встановлює базову тему застосунку.

### ChateeApplication.kt
Клас, що розширює `Application`. Позначений анотацією `@HiltAndroidApp`, що є обов'язковим для ініціалізації дерева залежностей **Hilt**. Тут починається життєвий цикл усіх Singleton-об'єктів (CryptoManager, Database тощо).

## Взаємодія з безпекою на рівні UI
UI-шар ніколи не отримує доступу до приватних ключів або зашифрованих байтів повідомлень. Завдяки архітектурі, ViewModel отримує від репозиторіїв вже готові до відображення доменні моделі (`Message`, `Chat`), де текст повідомлення вже дешифрований. Це гарантує, що логіка відображення залишається простою та захищеною від помилок у криптографічних розрахунках.
