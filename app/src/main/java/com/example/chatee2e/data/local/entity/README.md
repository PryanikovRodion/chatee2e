# Пакет: data.local.entity

Даний пакет містить опис моделей даних (сутностей), які визначають структуру таблиць у локальній реляційній базі даних SQLite. Кожен клас у цьому пакеті є відображенням бізнес-об'єкта на структуру бази даних Room.

## Основні сутності

### 1. ChatEntity
Визначає структуру таблиці `chats`, яка зберігає метадані про наявні діалоги.
* **Поля**:
    * `id`: унікальний ідентифікатор чату, що використовується для синхронізації з хмарним сховищем.
    * `name`: назва чату або ім'я співрозмовника для відображення в списку.
    * `participantsInfo`: рядок, що містить агреговану інформацію про учасників чату.
    * `lastMessageText` та `lastMessageTime`: денормалізовані поля для оптимізації рендерингу головного екрана (дозволяють не звертатися до таблиці повідомлень при кожному оновленні списку).

### 2. MessageEntity
Визначає структуру таблиці `messages`, призначеної для зберігання історії листування.
* **Поля**:
    * `chatId`: зовнішній ключ, що зв'язує повідомлення з об'єктом `ChatEntity`.
    * `text`: дешифрований вміст повідомлення, готовий до виводу на екран.
    * `timestamp`: часова мітка, що використовується для хронологічного сортування в межах чату.
    * `isMine`: прапорець, що визначає напрямок повідомлення (вхідне/вихідне) для логіки побудови UI.



## Роль у системі безпеки
Хоча сутності оперують відкритими даними (текстом повідомлень), фактична безпека на рівні диска забезпечується не кодом сутностей, а шаром шифрування SQLCipher, який інтегрується на рівні конфігурації бази даних.
